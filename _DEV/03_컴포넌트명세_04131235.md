#### 상태
| 상태 | 조건 | CSS |
|------|------|-----|

### C-13. Contact Form

#### 마크업
```html
<form id="contact-form" action="https://formspree.io/f/XXX" method="POST"
      novalidate aria-labelledby="contact-title">

  <!-- 허니팟 (봇 차단) -->
  <input type="text" name="_gotcha" tabindex="-1" autocomplete="off"
         class="sr-only" aria-hidden="true">

  <div class="field">
    <label for="name">이름<span aria-hidden="true" class="req">*</span></label>
    <input id="name" name="name" type="text" required minlength="2"
           autocomplete="name" aria-required="true" aria-describedby="name-error"
           class="form-input">
    <p id="name-error" class="field-error" role="alert" hidden></p>
  </div>

  <!-- 연락처, 이메일, 종류, 수량, 후가공, 요구사항, 동의 ... -->

  <div id="estimate-box" class="estimate-box" aria-live="polite" hidden>
    <span class="estimate-label">예상 견적</span>
    <strong id="estimate-amount" class="estimate-amount">₩0</strong>
  </div>

  <button type="submit" class="btn-primary w-full" aria-live="polite">
    <span class="btn-label">접수하기</span>
    <span class="btn-spinner" aria-hidden="true" hidden></span>
  </button>
</form>
```

#### 필드 검증 규칙
| 필드 | 규칙 | 에러 메시지 |
|------|------|-------------|
| `name` | `.trim().length >= 2` | 이름을 2자 이상 입력해 주세요. |
| `phone` | `/^01[0-9]-?\d{3,4}-?\d{4}$/` | 올바른 휴대폰 번호를 입력해 주세요. |
| `email` | `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` | 올바른 이메일 주소를 입력해 주세요. |
| `type` | `required` | 명함 종류를 선택해 주세요. |
| `quantity` | `required` | 수량을 선택해 주세요. |
| `agreement` | `.checked === true` | 개인정보 수집·이용에 동의해 주세요. |

#### 상태 머신
```
IDLE ──submit──→ VALIDATING
    ↓                ↓ (valid)
INVALID           SUBMITTING
    ↑                ↓
    └── invalid ──→  SUCCESS / ERROR
```

#### 접근성
- 모든 입력 `label[for]` 연결
- 에러 `role="alert"` + `aria-invalid="true"` + `aria-describedby`
- 제출 버튼 `aria-busy="true"` (로딩 시)
- 필수 표식 `<span aria-hidden="true" class="req">*</span>` (시각 전용)

#### JS API (개념)
```ts
interface CarouselAPI {
  goTo(index: number): void;
  next(): void;
  prev(): void;
  play(): void;          // 5초 자동 재생 시작
  pause(): void;
  destroy(): void;
}
```

#### 반응형 슬라이드 수
| 브레이크포인트 | 동시 노출 |
|---------------|-----------|
| < 768px | 1 |
| 768 ~ 1023px | 2 |
| ≥ 1024px | 3 |

#### 인터랙션
- 마우스 hover → `pause()`
- 마우스 leave → `play()`
- 터치 스와이프 50px 이상 → `prev/next`
- 도트 클릭 → `goTo(i) + 자동재생 리셋`
- 창 리사이즈 → 슬라이드 수 재계산

### C-15. Estimator Box

#### 마크업
```html
<div id="estimate-box" class="estimate-box" aria-live="polite" aria-atomic="true" hidden>
  <span class="estimate-label">예상 견적</span>
  <strong id="estimate-amount" class="estimate-amount">₩0</strong>
  <span id="estimate-discount" class="estimate-discount" hidden>대량 할인 10% 적용</span>
</div>
```

#### 가격 로직
```js
const BASE = { basic: 15000, premium: 25000, special: 40000 };
const ADDONS = { coating: 3000, gold: 10000, silver: 8000, emboss: 12000, diecut: 15000 };

function calculate({ type, qty, options }) {
  if (!type || !qty || qty === 'other') return null;

  const units = parseInt(qty, 10) / 100;           // 100매 단위
  const base = BASE[type] * units;
  const addons = options.reduce((sum, k) => sum + ADDONS[k] * units, 0);

  const rawTotal = base + addons;
  const discount = qty >= 1000 ? 0.10 : qty >= 500 ? 0.05 : 0;

  return { total: Math.round(rawTotal * (1 - discount)), discount };
}
```

#### 노출 규칙
- `total === null` → `hidden` 유지
- `total > 0` → 박스 노출 + `price-pop` 애니메이션
- `discount > 0` → `.estimate-discount` 텍스트 표시

## 4. 상태 표기 범례

각 컴포넌트의 상태는 다음 키로 통일합니다.

| 키 | 의미 |
|----|------|
| `default` | 초기 상태 |
| `hover` | 마우스 오버 |
| `focus-visible` | 키보드 포커스 |
| `active` / `is-active` | 선택됨 / 클릭됨 |
| `disabled` | 비활성 |
| `loading` / `is-loading` | 로딩 중 |
| `error` / `has-error` | 유효성 실패 |
| `success` | 성공 피드백 |
| `open` / `[open]` | 펼침 (details/dialog) |
