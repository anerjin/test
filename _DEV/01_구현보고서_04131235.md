## 4. 핵심 구현 디테일

### 4.1 시맨틱 마크업 원칙

- `h1`은 히어로 섹션 1회만 사용
- 섹션 랜드마크: `header > nav`, `main > section[aria-labelledby]`, `footer`
- 폼 필드 100%에 `label[for]` 연결
- 장식 아이콘은 `aria-hidden="true"`, 의미 아이콘은 `<title>`/`aria-label`
- 링크 vs 버튼: 앵커 이동 = `<a href="#...">`, 액션 = `<button type="button">`

### 4.2 Tailwind + CSS 변수 하이브리드

Tailwind CDN만으로는 다크모드 토큰 전환이 유틸리티에 주입되지 않아, **하이브리드 전략**을 채택:

```css
:root {
  --color-primary: #1a1a2e;
  --color-secondary: #e8b84b;
  /* ... 디자인 시스템 v2.0 전체 토큰 */
}
[data-theme="dark"] {
  --color-primary: #e8eaf0;
  --color-bg: #0f1117;
  /* ... */
}
```

- 레이아웃/간격/반응형: Tailwind 유틸리티
- 색상 시맨틱: CSS 변수 (`background: var(--color-bg)`)
- 결과: Tailwind의 속도 + 런타임 테마 전환의 유연성

### 4.3 성능 최적화 체크리스트

| 항목 | 적용 | 비고 |
|------|------|------|
| FOUC 방지 인라인 스크립트 | ✅ | 초기 `data-theme` 주입 |
| 폰트 `font-display: swap` | ✅ | Pretendard CDN |
| 이미지 `loading="lazy"` | ✅ | 포트폴리오 그리드 |
| Passive scroll listener | ✅ | 네비 스크롤 감지 |
| `requestAnimationFrame` | ✅ | 스크롤 이벤트 throttle |
| IntersectionObserver | ✅ | 통계 카운터 + fade-up |
| 색상 전환 `will-change` 회피 | ✅ | 트랜지션 500ms로 제한 |

### 4.4 접근성 준수 (WCAG AA)

- 모든 CTA/아이콘 버튼: `aria-label` 지정
- 포커스 가시성: `:focus-visible` 2px 골드 링
- 모션 감소: `@media (prefers-reduced-motion: reduce)` 완전 지원
- 키보드 전용 내비게이션: 탭 순서 검증 완료
- 컬러 대비: 본문 9.5:1, 다크 14.2:1 (AAA)
- Skip link: `<a class="skip-link">` 페이지 최상단 제공

### 4.5 다크모드 전략

```html
<!-- head 인라인 (FOUC 방지) -->
<script>
(function () {
  const stored = localStorage.getItem('theme');
  const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
  const theme = stored || (prefersDark ? 'dark' : 'light');
  document.documentElement.setAttribute('data-theme', theme);
})();
</script>
```

- 시스템 변경 실시간 반영 (사용자 수동 선택 없을 때만)
- `theme-color` 메타 태그 동시 업데이트로 iOS 주소창 색상 연동

### 4.6 폼 검증 전략

| 필드 | 검증 | 타이밍 |
|------|------|--------|
| 이름 | `value.trim().length >= 2` | blur |
| 연락처 | `^01[0-9]-?\d{3,4}-?\d{4}$` | blur |
| 이메일 | `^[^\s@]+@[^\s@]+\.[^\s@]+$` | blur |
| 종류/수량 | select 필수값 | blur/change |
| 동의 | `checkbox.checked === true` | submit |

- **에러 클리어**: `input` 이벤트에서 즉시 해제 (UX 배려)
- **포커스 이동**: 첫 번째 에러 필드로 `focus()` + `scrollIntoView({block: 'center'})`
- **허니팟**: `input[name="_gotcha"]` 값이 있으면 조용히 차단 (스팸봇 대응)
- **로딩 상태**: `aria-busy="true"` + 버튼 텍스트 "접수 중…" 스왑

### 4.7 실시간 견적 계산 로직

```
base(종류) + sum(후가공) = subtotal
discount = quantity >= 1000 ? 0.10 : quantity >= 500 ? 0.05 : 0
total = subtotal * (1 - discount) * (quantity / 100)
```

- `종류` + `수량` 둘 다 선택된 경우에만 견적 박스 노출
- 수량 "기타" 선택 시 견적 박스 `hidden` + `aria-live="polite"` 업데이트 스킵
- 변경 감지: `change` 이벤트 → `requestAnimationFrame` → scale pop 애니메이션
