## 1. 프로젝트 개요

### 1.1 비즈니스 목표

| 목표 | KPI | 측정 방법 |
|------|-----|-----------|
| 온라인 명함 제작 접수 증가 | 월 접수 건수 50건 이상 | 폼 제출 수 추적 |
| 회사 신뢰도 구축 | 이탈률 40% 이하 | Google Analytics |
| 모바일 접근성 확보 | 모바일 트래픽 비중 60%+ | UA 분석 |
| 빠른 정보 전달 | 평균 체류시간 2분+ | GA 체류시간 |

### 1.2 프로젝트 범위

- **형태**: 원페이지(Single Page) 반응형 웹사이트
- **핵심 기능**: 회사 소개 + 서비스 안내 + 명함 제작 접수 폼
- **기술 스택**: HTML + Tailwind CSS + Vanilla JS (경량 정적 사이트)
- **배포**: GitHub Pages 또는 Vercel (무료 정적 호스팅)

### 1.3 프로젝트 제약사항

- 서버 없이 정적 사이트로 구현 (폼 제출은 외부 서비스 활용)
- 원페이지 구조 (별도 라우팅 없음)
- 빠른 로딩 속도 우선 (3초 이내 FCP)

## 2. 사용자 분석

### 2.1 타겟 사용자 페르소나

#### 페르소나 A: 스타트업 대표 (Primary)

| 항목 | 내용 |
|------|------|
| **이름** | 김창업 (32세, 남) |
| **직업** | IT 스타트업 대표 |
| **목표** | 깔끔하고 세련된 명함을 빠르게 제작하고 싶다 |
| **Pain Point** | 오프라인 인쇄소 방문 시간 부족, 디자인 감각 부족 |
| **행동 패턴** | 모바일로 검색 → 빠르게 정보 파악 → 즉시 접수 |
| **기대** | 3분 이내 접수 완료, 시안 확인 후 결제 |

#### 페르소나 B: 중소기업 총무팀 담당자 (Secondary)

| 항목 | 내용 |
|------|------|
| **이름** | 박총무 (28세, 여) |
| **직업** | 중소기업 경영지원팀 |
| **목표** | 부서별 명함을 대량으로 일괄 주문하고 싶다 |
| **Pain Point** | 여러 직원 정보를 한 번에 전달하기 번거로움 |
| **행동 패턴** | PC로 검색 → 가격/납기 확인 → 상담 후 대량 주문 |
| **기대** | 명확한 가격 정보, 대량 주문 할인, 빠른 응답 |

#### 페르소나 C: 프리랜서 / 1인 사업자 (Tertiary)

| 항목 | 내용 |
|------|------|
| **이름** | 이프리 (35세, 남) |
| **직업** | 프리랜서 디자이너 |
| **목표** | 자신만의 개성 있는 명함을 소량 제작하고 싶다 |
| **Pain Point** | 소량 주문 가능 여부, 특수 용지/후가공 옵션 확인 |
| **행동 패턴** | 포트폴리오 참고 → 옵션 비교 → 커스텀 요청 |
| **기대** | 다양한 용지/후가공 옵션, 포트폴리오 확인 |

### 2.2 사용자 여정 맵 (User Journey Map)

```
[인지] → [방문] → [탐색] → [확신] → [접수] → [후속]

1. 인지: 검색엔진/SNS/지인추천으로 서비스 인지
2. 방문: 홈페이지 접속 → Hero 섹션에서 핵심 가치 확인
3. 탐색: 스크롤하며 서비스, 포트폴리오, 가격 확인
4. 확신: 제작 사례/고객 후기로 신뢰 형성
5. 접수: 명함 제작 접수 폼 작성 및 제출
6. 후속: 확인 메시지 → 담당자 연락 → 시안 제공 → 제작
```
